Scène 1

Intelligence (EN) = Renseignement

La CIA, c'est Central Intelligence Agency. Pas une agence de QI : une agence de renseignement. De donnée.

« L'IA, c'est pareil. Elle n'est pas intelligente. Elle est renseignée. La seule question qui compte aujourd'hui : renseignée avec les données de qui ? »

Si la donnée est le carburant, la vôtre a de la valeur. On parle confidentialité.

01

JE CHERCHE

Perplexity vs Gemini

Le besoin : « J'ai besoin d'une info fiable, et de pouvoir dire d'où elle vient. »

Perplexity — on veut vérifier

~94 %*

Cite ses sources phrase par phrase, cliquables. Pour benchmark, veille, réglementaire.

Gemini — on veut synthétiser & créer

~87 %*

Intégré à Google, plus orienté production dans la foulée.

Quand l'enjeu monte — les modes avancés

  • Pro Search (Perplexity) : pose des questions, lit des dizaines de sources, raisonne en étapes.
  • Deep Research (Perplexity ET Gemini) : 2-4 min, des centaines de pages lues, un rapport structuré en sortie. Plusieurs heures de travail humain condensées.
Données publiques uniquement — aucun chiffre interne dans la requête. « Perplexity ne vous dispense pas de vérifier. Il vous dit OÙ vérifier. On clique. »

* Ordres de grandeur (comparatifs publics 2026), pas une garantie.

« J'ai mon info. Mais j'ai 10 documents à digérer. Étape suivante. »

02

JE COMPRENDS

NotebookLM

Le besoin : « La vérité est dans MES documents, pas sur Google. »

NotebookLM (Google, propulsé par Gemini) ne travaille que sur les sources que vous chargez. Chaque réponse est ancrée, avec le passage exact cité. Il n'invente pas.

« C'est l'outil du "j'ai trop à lire". »

Ce qu'il sort de votre corpus, en un clic (Studio)

  • Q&R sourcée sur vos docs
  • Audio Overview — deux voix IA discutent vos documents. Un dossier digéré en marchant.
  • Mind Map interactive — une branche cliquée ouvre un chat ciblé
  • Video, slides, infographies, quiz, flashcards
Outil grand public → documents non sensibles. Du sensible interne → Copilot / SharePoint (M365).

« J'ai compris. Maintenant je produis. »

JE PRODUIS

Le besoin change selon ce qu'on fabrique

Un texte → Copilot Word M365

  • Générer / Réécrire / Résumer
  • Il lit le doc ouvert — « à partir de ce document », pas de copier-coller

Des chiffres → Copilot Excel M365

  • D'abord : données en tableau (Insertion → Tableau). Sinon rien ne marche. Le point bloquant.
  • « 3 boutiques en plus forte progression » · « graphique du CA par catégorie » · « marge %, rouge sous 20 % »
  • Demande la formule : tu comprends, tu réutilises

Une image → Gemini / Nano Banana Pro Wksp · Marketing

Génération et édition. Sa force : texte intégré lisible, jusqu'à 4K.

Sujet+ Style+ Cadrage+ Lumière+ Ambiance+ Format
  • Texte dans l'image : entre guillemets + position. « …texte "Édition Voyage" centré en haut. »
  • Modifier sans tout refaire : « Garde la compo, change le texte en 'Nouvel An'. » → une modif à la fois
CA réel → Copilot M365, jamais d'export. Projet visuel réel → Gemini Workspace.

« C'est produit. Il faut le faire circuler. »

Aparté · le niveau au-dessus du « je produis »

La puissance de créer en HTML avec Gemini

« Décrivez le livrable — Gemini Canvas écrit le code. Vous récupérez un objet fini, pas un brouillon à mettre en forme. »

Présentation interactive

Navigation, animations, quiz intégrés — ce qu'un PowerPoint ne fait pas sans plugin ni licence.

Document charté

Vos couleurs, vos typos, votre logo appliqués partout, automatiquement. La charte devient un prompt.

One-pager · infographie

Un récap visuel responsive : lisible sur le téléphone d'un concédant comme sur l'écran d'un CODIR.

Mini-site · quiz

Un lien à partager. Rien à installer : ça s'ouvre dans n'importe quel navigateur.

Pourquoi ça marche

  • Un seul fichier autonome — pas de dépendances, pas de logiciel côté lecteur
  • Responsive par nature — le contenu se réorganise seul, mobile → grand écran
  • Itérable à l'infini — « change la couleur », « ajoute une section », « plus aéré »

Le prompt type

« Crée une page HTML responsive. Charte : [couleurs, typos]. Structure : [vos sections]. Navigation sticky, design moderne. »

Puis on itère, une demande à la fois — exactement comme pour une image.

La preuve ? Ce support que vous lisez est exactement ça — un fichier HTML unique, généré et itéré avec une IA.

Charte et contenus internes dans le prompt → Gemini Workspace. Livrable diffusé à l'extérieur = contenu non sensible.

Le cœur du Bloc 3

Les Gems Gemini

Un Gem = un assistant que vous fabriquez : Nom + Instructions + outil + fichiers de connaissance. Chaque conversation rejoue le cadrage, sans rien recoller. Gratuits sur Gemini, partageables dans l'organisation (Workspace).

Démo — le Gem « Réponse concédant »

  • Rôle + ton Translia
  • Règles permanentes
  • Modèles + lexique en connaissance
  • On relance : tout est déjà là

Autres Gems utiles

CR structuréSynthèse de reportingVeille retail

Garde-fou non négociable

Validation humaine sur tout ce qui touche données sensibles ou envoi externe. Principe d'un agent : on délègue une procédure, l'IA l'exécute sous supervision.

Documents internes dans un Gem → Workspace Translia uniquement. Flux automatisé sur données Translia → périmètre validé DSI.

Atelier

Chacun configure un Gem réel (cas non sensible). Partage des cadrages.

Objectif de sortie

Repartir avec au moins un Gem opérationnel. Engagement : « Le Gem que je réutilise dès cette semaine. »

Bloc 3 · côté Microsoft

Les agents Copilot — le pendant M365 des Gems

Même logique : on configure une fois — instructions + connaissances + actions — et l'assistant rejoue dans Teams et Copilot Chat. Création guidée en langage naturel (agent builder / Copilot Studio), partageable dans l'organisation.

Gem Gemini

  • Connaissance : fichiers chargés dans le Gem
  • Vit dans Gemini · partage via Workspace
  • Docs internes → Workspace uniquement

Agent Copilot

  • Connaissance : SharePoint, sites, fichiers M365 — déjà dans le périmètre
  • Vit dans Teams & Copilot Chat · partage dans l'organisation
  • Atout décisif : données internes OK d'office — le contrat Translia couvre tout

Trois agents utiles chez Translia

  • « FAQ procédures » — branché sur le SharePoint de service : les réponses citent le document interne, fini de chercher
  • « Réponse concédant » — version M365 du Gem du même nom : modèles + ton maison, nominatif autorisé
  • « Synthèse de reporting » — cadrage permanent, chacun y dépose son fichier du mois

Le bon choix reste le réflexe du Bloc 1 : quel compte, quelle donnée ? Donnée interne → agent Copilot. Donnée publique, besoin créatif → Gem.

Création selon votre licence Copilot — certains droits d'agent builder sont ouverts ou non par la DSI. Déploiement d'un agent à l'échelle d'une équipe → validation DSI, comme tout flux automatisé.
